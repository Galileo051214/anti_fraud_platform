## 1. 项目概述

**反诈骗学习平台**面向高校场景，提供反诈资讯与案例学习、游戏化闯关与情景模拟（FSM）、社区互动、智能客服（DeepSeek 大模型）、混合推荐算法与数据统计等能力。系统采用前后端分离架构：**Spring Boot 3** 提供 REST API，**Vue 3 + Vite** 提供单页应用；数据持久化使用 **MySQL**，会话与缓存使用 **Redis**；接口文档使用 **SpringDoc (OpenAPI 3)**。

### 3.2 Spring Security + JWT

- 关闭 CSRF，采用 **STATELESS** 会话；在 `UsernamePasswordAuthenticationFilter` 之前挂载 **JWT 过滤器**。
- 匿名放行注册、登录、静态上传资源、Swagger 等路径；`/api/admin/**` 需 `ADMIN` 角色。

```31:60:e:\2026work\4547\backend\src\main\java\com\anti\config\SecurityConfig.java
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)
            .cors(cors -> {})
            .sessionManagement(session -> session
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            // ...
            .authorizeHttpRequests(auth -> auth
                .requestMatchers(
                    "/api/user/register",
                    "/api/user/login",

                ).permitAll()
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
```

```31:42:e:\2026work\4547\backend\src\main\java\com\anti\security\JwtUtils.java
    public String generateToken(String username, String role, Long userId) {
        Date now = new Date();
        Date expiryDate = new Date(now.getTime() + expiration);

        return Jwts.builder()
                .subject(username)
                .claim("role", role)
                .claim("userId", userId)

                .expiration(expiryDate)
                .signWith(key)
                .compact();
```

### 3.3 混合推荐算法（示例：威尔逊置信度）

`RecommendationAlgorithmUtil` 实现威尔逊得分、余弦相似度、皮尔逊相关系数等，用于案例/内容排序与用户相似度计算。

```20:41:e:\2026work\4547\backend\src\main\java\com\anti\util\RecommendationAlgorithmUtil.java
    public static BigDecimal calculateWilsonScore(int positive, int total) {
        if (total == 0) {
            return BigDecimal.ZERO;
        }
        double p = (double) positive / total;
        double z2 = Z * Z;
        double n = total;

        double denominator = 1 + z2 / n;
        double sqrtTerm = Math.sqrt((p * (1 - p) / n) + (z2 / (4 * n * n)));
        double score = (p + z2 / (2 * n) - Z * sqrtTerm) / denominator;

        return BigDecimal.valueOf(score).setScale(4, RoundingMode.HALF_UP);
    }
```

推荐服务接口按用户生命周期划分策略（新手冷启动、成长期、成熟期），见 `RecommendationService` 与 `RecommendationServiceImpl`。

### 3.4 前端 Axios 与路由鉴权

开发环境下，`vite.config.ts` 将 `/api` 代理到 `http://localhost:8080`，与后端 `server.port` 一致。请求封装自动附加 `Authorization: Bearer <token>`：

```7:21:e:\2026work\4547\frontend\src\utils\request.ts
const service: AxiosInstance = axios.create({
  baseURL: '/api',
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json;charset=utf-8'
  }
})

service.interceptors.request.use(
  (config: InternalAxiosRequestConfig) => {
    const token = localStorage.getItem('token')
    if (token && config.headers) {
      config.headers['Authorization'] = `Bearer ${token}`
    }
    return config
  },
```

## 5. 系统部署与使用说明

### 5.1 环境要求

- **JDK** 17+
- **Maven** 3.8+（用于后端构建）
- **Node.js** 18+ 与 **npm**（或 pnpm/yarn，与团队规范一致）
- **MySQL** 8.x（推荐 `utf8mb4`）
- **Redis** 6+

### 5.2 数据库初始化

1. 在 MySQL 中执行 `sql/init.sql`，创建库 `anti_fraud_platform` 及表结构。
2. 按需执行 `sql/seed_case.sql`、`sql/seed_news.sql`、`sql/seed_challenge.sql` 等导入演示数据。
3. 若存在增量脚本（如 `patch_user_last_login.sql`），在对应版本升级时执行。

### 5.3 后端配置与启动

1. 默认激活 **`dev`** 配置（见 `application.yml` 中 `spring.profiles.active`）。
2. 在 `application-dev.yml` 中配置本地 **数据源**、**Redis**、**JWT** 密钥长度（需满足 HMAC 要求）等。
3. **生产环境**建议使用 `application-prod.yml`，通过环境变量注入敏感信息，例如：
   - `DB_USERNAME`、`DB_PASSWORD`
   - `REDIS_HOST`、`REDIS_PORT`、`REDIS_PASSWORD`
   - `JWT_SECRET`
4. **第三方 AI**：DeepSeek 相关 URL、API Key、模型名等应通过配置注入；**切勿将真实密钥提交到版本库或写入对外文档**。生产部署请使用环境变量或密钥管理服务。
5. 在项目根目录执行：

```bash
cd backend
mvn spring-boot:run -DskipTests
或
mvn -q -DskipTests package
java -jar target/anti-fraud-platform-1.0.0.jar
```

（具体 jar 名称以 `pom.xml` 中 `artifactId` 与 `version` 为准。）\
6\. 服务默认端口 **8080**（见 `application.yml`）。\
7\. 开发环境下可访问 **Swagger UI**（路径见 `springdoc` 与 `SecurityConfig` 白名单，如 `/swagger-ui.html`）。

### 5.4 前端配置与启动

1. 安装依赖并启动开发服务器：

```bash
cd frontend
npm install
npm run dev
```

1. 默认开发端口 **5173**，`vite.config.ts` 中已将 **`/api`** **代理到** **`http://localhost:8080`**，需先启动后端。
2. 生产构建：

```bash
npm run build
```

将 `frontend/dist` 静态资源部署到 **Nginx** 等；需将 **`/api`** **反向代理**到后端服务（例如 `proxy_pass http://127.0.0.1:8080`），并配置 **HTTPS**、缓存策略等。

### 5.5 典型使用流程（终端用户）

1. 访问前端地址，**注册/登录**（学生或管理员角色由业务与种子数据决定）。
2. 浏览 **首页、资讯、案例**；参与 **闯关、情景模拟**；使用 **社区** 发帖评论；通过 **智能客服** 提问。
3. **个人中心**维护资料；**积分/成就** 查看 gamification 反馈；**智能推荐** 页查看个性化内容。
4. 管理员登录后可进入 **`/admin`** 子路由对应的管理功能（需后端返回 `admin` 角色）。
